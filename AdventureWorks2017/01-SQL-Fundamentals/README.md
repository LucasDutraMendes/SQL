# AdventureWorks2017

This section contains SQL exercises and projects developed using the **AdventureWorks2017** sample database for Microsoft SQL Server.

The exercises are organized by topic, progressing from fundamental SQL concepts to advanced querying and performance optimization.

## Modules

| Module | Description |
|---------|-------------|
| **01 - SQL Fundamentals** | Basic SQL syntax including `SELECT`, `WHERE`, `ORDER BY`, `TOP`, `DISTINCT`, `LIKE`, `BETWEEN`, `IN`, and `NULL` handling. |
| **02 - Functions** | String, numeric, date, conversion, and conditional functions such as `CASE`, `CAST`, `CONVERT`, `ISNULL`, and `COALESCE`. |
| **03 - Joins** | Combining data from multiple tables using `INNER`, `LEFT`, `RIGHT`, `FULL`, `CROSS`, and `SELF JOIN`. |
| **04 - Grouping & Aggregation** | Aggregating data using `GROUP BY`, `HAVING`, and aggregate functions (`COUNT`, `SUM`, `AVG`, `MIN`, `MAX`). |
| **05 - Subqueries** | Scalar, correlated, and nested subqueries using `IN`, `EXISTS`, `ANY`, and `ALL`. |
| **06 - Common Table Expressions (CTEs)** | Organizing complex queries with Common Table Expressions, including recursive CTEs. |
| **07 - Window Functions** | Analytical functions such as `ROW_NUMBER`, `RANK`, `DENSE_RANK`, `LAG`, `LEAD`, and window aggregates. |
| **08 - T-SQL Programming** | Variables, control-of-flow statements, temporary tables, table variables, and error handling. |
| **09 - Views** | Creating and managing database views for abstraction and reusable queries. |
| **10 - Stored Procedures** | Building reusable stored procedures with parameters and execution logic. |
| **11 - Indexes** | Clustered and nonclustered indexes, execution plans, and indexing strategies. |
| **12 - Query Performance** | Query optimization techniques, SARGability, execution plans, statistics, and performance tuning. |
| **Challenges** | Business-oriented SQL problems combining concepts from multiple modules. |

## Learning Goals

- Build strong SQL fundamentals.
- Write clean, readable, and maintainable queries.
- Develop analytical thinking for real-world business scenarios.
- Learn SQL Server best practices.
- Prepare for Data Engineering technical interviews.
